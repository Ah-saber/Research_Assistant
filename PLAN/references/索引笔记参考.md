# 索引笔记参考

本文档提供索引笔记的格式规范和最佳实践。索引笔记用于组织和链接相关主题的笔记。

## 核心原则

1. **使用表格格式**：使用Markdown表格组织笔记，便于Dataview插件自动索引
2. **添加说明列**：每条笔记都应有简短的说明
3. **分类组织**：按主题或时间顺序分章节组织
4. **添加相关主题**：在索引末尾添加相关主题链接
5. **更新记录**：记录索引的创建和修改历史

## 完整模板

```yaml
---
title: "主题索引"
type: index
tags: [日志, 主题, 索引]
date: YYYY-MM-DD
related: [["相关主题1"], ["相关主题2"]]
---

# 主题索引

本索引整合了主题相关的所有笔记。

## 1. 分类1

| 笔记 | 说明 |
|------|------|
| [[笔记名称]] | 笔记的简短说明 |

## 2. 分类2

| 时间/类别 | 笔记 |
|----------|------|
| 时间/类别名 | [[笔记名称]] |

## 相关主题

- [[相关主题]] - 说明

## 更新记录

- YYYY-MM-DD: 创建索引笔记
```

## 格式规范

### Frontmatter

```yaml
---
title: "主题索引"        # 索引标题，以"索引"结尾
type: index             # 类型固定为index
tags: [日志, 主题, 索引]  # 包含"索引"标签
date: YYYY-MM-DD        # 创建日期
related: [["主题"]]      # 相关主题（可选）
---
```

### 内容结构

1. **简介段落**：说明本索引的范围和目的
2. **分类章节**：按逻辑分组织笔记，每个分类使用二级标题（##）
3. **表格格式**：使用Markdown表格，至少包含"笔记"和"说明"两列
4. **相关主题**：列出与本索引相关的其他主题索引
5. **更新记录**：记录索引的创建和修改历史

### 表格格式

**标准格式**：
```markdown
| 笔记 | 说明 |
|------|------|
| [[笔记名称]] | 笔记说明 |
```

**带时间的格式**：
```markdown
| 时间 | 笔记 |
|------|------|
| 2024年1月 | [[笔记名称]] |
```

**三列格式**：
```markdown
| 周次 | 笔记 | 说明 |
|------|------|------|
| 第11周 | [[周报名称]] | 说明 |
```

### 链接格式

- **相对路径**：使用`[[目录/笔记名称]]`格式链接子目录中的笔记
- **相关主题**：使用`[[日志/主题]]`格式链接其他主题索引
- **外部链接**：使用`[[链接|描述]]`格式添加描述性链接

## 命名规范

- 索引笔记文件名：`00-主题索引.md`
- 索引笔记标题：`# 主题索引`
- 放在对应目录的根目录下

## 最佳实践

1. **定期更新**：添加新笔记时及时更新索引
2. **保持一致**：使用统一的格式和命名规范
3. **添加说明**：每条笔记都应有简短的说明，便于快速理解
4. **相关链接**：在索引末尾添加相关主题链接，便于跳转
5. **更新记录**：记录索引的修改历史，便于追溯

## Dataview集成

如果使用Dataview插件，可以在索引中添加自动查询：

```markdown
## 自动索引

```dataview
LIST
FROM "日志/实验室"
WHERE !contains(file.path, "00-")
SORT file.ctime DESC
```
```

## 注意事项

1. **不要过度索引**：只在有足够多相关笔记时创建索引
2. **避免重复**：检查是否已有类似索引，避免重复创建
3. **保持简洁**：索引应简洁明了，不要包含过多内容
4. **链接有效**：确保所有链接都能正常工作
