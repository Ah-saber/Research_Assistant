# Note Analyze Skill

分析Obsidian笔记的结构和内容。

## 使用场景

当用户说：
- "分析笔记"
- "笔记结构分析"
- "查看笔记统计"
- "分析[目录]的笔记"

## 工作流程

1. **确定分析范围**
   - 如果指定目录：分析该目录下的所有笔记
   - 如果没有指定：询问用户分析范围

2. **扫描笔记**
   - 使用Glob工具查找.md文件
   - 避开IDEA和思绪目录（除非授权）

3. **分析内容**
   - 统计笔记数量
   - 分析目录结构
   - 提取标签使用情况
   - 检测Wikilinks使用情况
   - 分析frontmatter使用情况
   - 识别孤立笔记（没有链接的笔记）

4. **生成报告**
   - 结构化展示分析结果
   - 提供改进建议

## 示例输出

```markdown
# 笔记分析报告

## 基本统计
- **总笔记数**: 156
- **目录数**: 12
- **标签数**: 48
- **链接数**: 234

## 目录结构
```
科研/
├── 论文/ (45篇)
├── 概念/ (32篇)
├── 方法/ (28篇)
└── 项目/ (15篇)
```

## 标签分析
最常用标签：
| 标签 | 使用次数 |
|------|----------|
| #论文/深度学习 | 28 |
| #论文/NLP | 23 |
| #概念/模型 | 19 |

## 链接分析
- **有出站链接的笔记**: 89 (57%)
- **有入站链接的笔记**: 67 (43%)
- **孤立笔记**: 34 (22%)

## 改进建议
1. 34篇笔记没有链接，可以考虑建立关联
2. 标签使用不够一致，建议统一标签体系
3. 部分笔记缺少frontmatter元数据
```

## 相关命令

- `/note-link` - 建立笔记关联
- `/note-template` - 创建笔记模板
