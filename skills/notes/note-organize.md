# Note Organize Skill

整理和优化Obsidian笔记。

## 使用场景

当用户说：
- "整理笔记"
- "优化笔记"
- "重新组织笔记"

## 工作流程

1. **分析现状**
   - 使用`/note-analyze`分析当前笔记结构
   - 识别问题和改进空间

2. **提出方案**
   - 展示整理建议
   - 等待用户确认

3. **执行整理**
   - 重命名文件（如果需要）
   - 调整目录结构
   - 统一标签格式
   - 添加缺失的frontmatter
   - 合并重复笔记

## 注意事项
- 先展示方案，不要直接修改
- 尊重用户现有笔记结构
- 只做必要的调整
