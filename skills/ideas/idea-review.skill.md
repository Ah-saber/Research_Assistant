# /idea-review - 回顾和管理Idea

## 触发词
- "回顾Idea"
- "查看想法"
- "Idea回顾"
- "查看Inspiration"
- "我的想法"

## 功能描述
回顾Inspiration目录中的Idea，按不同维度查看和评估，提供行动建议。

## 回顾方式

### 1. 按状态筛选

| 状态 | 说明 |
|------|------|
| sprout | 萌芽状态的想法 |
| thinking | 正在思考的Idea |
| implemented | 已实现的Idea |
| abandoned | 已放弃的Idea |

### 2. 按时间排序
- 最新创建
- 最近更新
- 长期未更新

### 3. 按主题分组
- 深度学习
- 实验想法
- 应用场景
- 理论思考
- 工具改进

### 4. 按优先级
- high：高优先级
- medium：中优先级
- low：低优先级

## 回顾内容

### 基本信息
- Idea标题
- 创建时间
- 当前状态
- 优先级
- 主题标签

### 关联内容
- 相关笔记
- 相关论文
- 相关Idea

### 行动建议
根据Idea状态和内容提供建议：
- 阅读建议：推荐阅读的论文或笔记
- 实验建议：可以尝试的实验
- 笔记建议：可以创建的相关笔记
- 状态更新：建议推进或搁置

## 回顾格式

```markdown
# Idea回顾报告

## 概览
- 总数：25个
- 本月新增：3个
- 待推进：10个

   - 相关：[[Transformer架构]], [[CNN可视化]]
   - 建议：调研相关论文，评估可行性

2. ...

### 思考中（8个）
...

### 已实现（3个）
...

### 已放弃（2个）
...

## 长期未更新（超过30天）
1. GNN分子结构 (2023-12-01)
   - 建议：评估是否继续推进或标记为已放弃

## 高优先级Idea（5个）
...

## 行动建议汇总
- 阅读论文：5篇推荐
- 实验验证：3个想法可实验
- 笔记整理：2个需要细化

- 输入序号查看详情
- 输入序号+状态更新状态

```

## 输入参数

| 参数 | 说明 | 必填 |
|------|------|------|
| filter | 筛选条件 | 否，默认全部 |
| sort | 排序方式 | 否，默认时间 |
| status | 状态筛选 | 否 |
| limit | 显示数量 | 否，默认20 |

## 行动建议生成

### 阅读建议
```
当Idea包含未读相关工作时：
→ 建议：阅读[[论文X]]以了解该方法
→ 预期：更好地评估Idea可行性
```

### 实验建议
```
当Idea是实验类且状态为思考中时：
→ 建议：设计小规模实验验证
→ 方向：对比baseline效果
→ 预期：初步验证想法有效性
```

### 笔记建议
```
当Idea相关但缺少详细笔记时：
→ 建议：创建详细笔记记录思考过程
→ 内容：动机、方法、预期结果
```

### 状态更新建议
```
根据Idea状态和时间：
- 萌芽超过30天 → 评估是否推进
- 思考中超过60天 → 决定实施或搁置
- 长期无更新 → 确认是否仍相关
```

## 与IDEA目录的关系

**重要**：
- `/idea-review` 仅回顾Inspiration目录
- 建议用户将已实现或有价值的Idea移至IDEA目录
- 不访问IDEA目录（除非用户授权）

## 移至IDEA建议

回顾时提示：
```
以下Idea建议移至科研/IDEA：
1. 已实现的Idea（3个）
2. 高价值思考中Idea（2个）

移至IDEA前请确保：
- [ ] 内容完整
- [ ] 有详细思考
- [ ] 有明确价值

手动移至：科研/IDEA/相应主题目录
```

## 注意事项

1. 不修改Idea状态（除非用户明确指示）
2. 建议仅供参考
3. 保留用户决策权
4. 不访问IDEA目录

## 与 Bases 仪表盘的关系

**三层协作架构**：

| 层级 | 组件 | 作用 |
|------|------|------|
| **格式参考** | obsidian-bases skill | 提供 Bases (.base) 格式规范文档 |
| **执行工具** | Read/Write/Edit + Glob | 底层文件操作 |
| **工作流封装** | /idea-tracker 等 | 业务逻辑 |

**分工明确**：
- **obsidian-bases**：提供 Bases 格式规范文档
  - 视图类型（table、cards、list、map）
  - 过滤条件、公式计算、分组排序、统计汇总
- **Dashboard Skills**：提供仪表盘工作流
  - `/idea-tracker` - Idea状态管理追踪
  - `/research-dashboard` - 综合研究进度仪表盘
- **/idea-review**：回顾Idea后，建议生成对应仪表盘

**不是重复造轮子**：
- obsidian-bases 是**文档技能**，不执行文件操作
- Dashboard Skills 是**工作流封装**，使用 Read/Write/Edit + Glob 创建Base文件
- 本技能是**回顾层**，识别需要追踪的Idea并建议调用对应仪表盘技能

**仪表盘建议触发条件**：
- Idea数量较多（>10个）→ 建议调用 `/idea-tracker` 生成状态追踪仪表盘
- 需要全景视图 → 建议调用 `/research-dashboard` 生成综合研究进度仪表盘
- 需要按主题/状态/优先级筛选 → 建议使用仪表盘的多视图功能
