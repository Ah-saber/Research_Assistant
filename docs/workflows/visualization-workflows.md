# 可视化工作流

本文档描述使用Research Assistant的可视化功能，包括Canvas图谱和Bases仪表盘。

---

## Canvas可视化工作流

### 工作流1：论文引用关系图谱 (`/paper-graph`)

生成展示论文之间引用和关联关系的可视化图谱。

**使用场景**：
- 了解研究领域的发展脉络
- 发现论文之间的传承关系
- 可视化研究主题的演进

**工作流步骤**：

```bash
/paper-graph
```

**选择图谱类型**：

1. **直接引用图谱**：展示论文之间的直接引用关系
2. **主题关联图谱**：展示按研究主题分组的论文关联
3. **时间演进图谱**：按时间顺序展示论文发展脉络

**输出**：
- `.canvas` 格式的可视化文件
- 层次布局，清晰展示引用方向
- 颜色编码：红色（核心）、橙色（重要）、黄色（扩展）、绿色（应用）

**示例**：
```
用户: /paper-graph

你: 正在生成论文引用关系图谱...

选择图谱类型：
1. 直接引用图谱
2. 主题关联图谱
3. 时间演进图谱

用户: 1

你: ✓ 已生成论文引用图谱
文件：Paper_引用关系.canvas

包含12篇论文，15条引用关系
```

### 工作流2：Idea概念关系图谱 (`/idea-map`)

生成展示Idea之间概念关联的可视化图谱。

**使用场景**：
- 浏览所有Idea的整体分布
- 发现相关Idea之间的联系
- 按状态或优先级管理Idea

**工作流步骤**：

```bash
/idea-map
```

**选择视图类型**：

1. **状态分组视图**：按萌芽/思考中/已实现/已放弃分组
2. **概念关联视图**：显示Idea之间的语义关联
3. **优先级视图**：按高/中/低优先级着色

**输出**：
- `.canvas` 格式的可视化文件
- 力导向布局，相关节点自然聚集
- 颜色编码：黄色（萌芽）、橙色（思考中）、绿色（已实现）、灰色（已放弃）

**与 `/idea-review` 结合**：
```bash
# 先生成图谱查看全貌
/idea-map

# 再按状态回顾
/idea-review --status sprout
```

---

### 工作流3：综合知识画布 (`/knowledge-canvas`)

整合论文、Idea、概念笔记的综合知识视图。

**使用场景**：
- 构建研究领域全景图
- 整合不同类型的知识
- 发现跨领域的关联

**工作流步骤**：

```bash
/knowledge-canvas
```

**选择内容类型**：

可包含以下内容：
- 论文笔记
- Idea笔记
- 概念笔记
- 项目笔记
- 问题笔记

**输出**：
- `.canvas` 格式的综合画布
- 区域布局：论文区、Idea区、概念区、问题区
- 支持拖拽调整和手动编辑

## Bases仪表盘工作流

### 工作流4：论文阅读进度追踪 (`/paper-dashboard`)

生成追踪论文阅读进度的Bases仪表盘。

**使用场景**：
- 追踪论文阅读状态
- 按领域或时间线统计阅读进度
- 管理待阅读论文列表

**工作流步骤**：

```bash
/paper-dashboard
```

**可用视图**：

| 视图 | 功能 |
|------|------|
| Table视图 | 详细表格，支持分组、排序、筛选 |
| Cards视图 | 卡片浏览，适合查看概览 |
| List视图 | 简洁列表，快速浏览 |

**公式功能**：
- 状态图标转换（状态 → emoji）
- 日期计算（距今天数、相对时间）
- 统计汇总（数量、平均值）

**与 `/paper-notes` 结合**：
```bash
# 创建论文笔记
/paper-notes "Attention Is All You Need"

# 更新仪表盘
/paper-dashboard
```

---

### 工作流5：Idea状态管理追踪 (`/idea-tracker`)

生成追踪Idea状态的Bases仪表盘。

**使用场景**：
- 追踪Idea进展状态
- 识别需要关注的内容
- 按主题管理Idea

**工作流步骤**：

```bash
/idea-tracker
```

**可用视图**：

| 视图 | 功能 |
|------|------|
| 按状态分组 | 萌芽/思考中/已实现/已放弃 |
| 按主题分组 | 同类Idea聚集显示 |
| 优先级视图 | 高优先级置顶显示 |

**过滤条件**：
- 仅显示活跃Idea
- 显示长期未更新的Idea
- 显示特定状态的Idea

**与 `/idea-review` 结合**：
```bash
# 先回顾Idea
/idea-review

# 再更新追踪仪表盘
/idea-tracker
```

---

### 工作流6：综合研究进度仪表盘 (`/research-dashboard`)

整合论文、Idea、概念、项目等多种内容类型的综合研究进度仪表盘。

**使用场景**：
- 获取研究进度全景
- 多维度追踪研究进展
- 生成研究进度报告

**工作流步骤**：

```bash
/research-dashboard
```

**内容类型**：

| 类型 | 说明 |
|------|------|
| 论文 | 已阅读、待阅读论文 |
| Idea | 各状态Idea统计 |
| 概念 | 已掌握概念 |
| 项目 | 进行中项目 |
| 实验 | 实验记录 |

**统计功能**：
- 各类型数量统计
- 按时间线统计
- 完成率计算

## 技能整合关系

### 三层架构

```
┌─────────────────────────────────────────┐
│         工作流封装（Skills）              │
│  /paper-graph /idea-map /knowledge-canvas│
│  /paper-dashboard /idea-tracker          │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│       格式规范（Reference Skills）        │
│  json-canvas skill  →  Canvas格式文档    │
│  obsidian-bases skill → Bases格式文档    │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│         执行工具（Tools）                 │
│  Read / Write / Edit / Glob              │
└─────────────────────────────────────────┘
```

**说明**：
- **工作流封装**：组合底层工具实现业务逻辑
- **格式规范**：提供Canvas和Bases格式参考文档
- **执行工具**：实际创建和修改文件

### 触发条件

| 技能 | 触发词 | 适用场景 |
|------|--------|----------|
| `/paper-graph` | "论文图谱"、"论文关系图" | 需要可视化论文关系 |
| `/idea-map` | "Idea图谱"、"想法地图" | 需要浏览Idea全貌 |
| `/knowledge-canvas` | "知识画布"、"研究图谱" | 需要综合知识视图 |
| `/paper-dashboard` | "论文仪表盘"、"阅读进度" | 需要追踪阅读进度 |
| `/idea-tracker` | "Idea追踪"、"想法管理" | 需要管理Idea状态 |
| `/research-dashboard` | "研究仪表盘"、"进度总览" | 需要综合进度视图 |

### 最佳实践

1. **结合使用**：可视化与基础技能结合，如 `/paper-notes` + `/paper-graph`
2. **定期更新**：定期运行仪表盘技能，保持进度追踪
3. **手动调整**：生成的Canvas和Bases可以手动编辑和调整
4. **版本控制**：将生成的可视化文件纳入git版本控制
